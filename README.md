## 📋 项目概述

**Deyes** 是一个基于 AI 技术的跨境电商数字员工系统，通过自动化选品、图片生成、商品上架、客服等环节，帮助卖家实现规模化运营。

### 核心能力

- **快速测款**: 日测试 1,200+ SKU，快速找到爆款
- **智能图片生成**: 自动复刻爆款风格，生成主图和详情页
- **A/B测试驱动**: 数据驱动优化，爆款率提升 3倍
- **多平台支持**: Temu, AliExpress, Amazon, Ozon, Rakuten, Mercado Libre
- **全链路自动化**: 从选品到上架的完整自动化

### 关键指标

| 指标 | 数值 |
|------|------|
| **日测款能力** | 1,200个SKU |
| **主图生成速度** | 60秒/SKU（3个变体） |
| **图片质量评分** | 92分（专业级） |
| **风格一致性** | 90%+ |
| **产品变形率** | <5% |
| **月爆款数量** | 5,400个（vs 传统15个） |
| **成本降低** | 90% |

### 核心组件

#### 图像生成层
- **FLUX.1-dev (FP8)**: 基础文生图模型（13GB）
- **Turbo LoRA**: 3倍速度提升
- **IPAdapter Plus**: 风格迁移，爆款复刻
- **ControlNet**: 防止产品变形（Canny + Depth）
- **FLUX Fill**: 局部编辑，背景替换
- **RMBG 1.4**: 自动背景移除

#### LLM推理层
- **SGLang**: 推理引擎（比vLLM快29%）
- **Qwen3.5-35B-A3B**: 多模态LLM（15.4GB/卡）
- **Qwen2-VL**: 图像理解和质量检测

#### 高级功能
- **IDM-VTON**: 虚拟试穿（服装类目）
- **风格库系统**: 10,000+ 风格自动匹配
- **A/B测试**: 数据驱动优化

### GPU资源分配（8x RTX 4090）

```
卡0-1: ComfyUI实例1 (主图生成)
  └─ FLUX.1 + IPAdapter + ControlNet + Turbo LoRA (~24GB)

显存利用率: 95%
```

## 🎯 业务模式

### 快速测款流程

```
1. 选品阶段
   ├─ 从1688抓取产品信息
   ├─ 分析爆款趋势
   └─ 推荐高潜力SKU

2. 图片生成阶段（60秒/SKU）⭐
   ├─ 背景移除（RMBG 1.4）
   ├─ 风格匹配（自动从风格库）
   ├─ 生成3个主图变体（A/B测试）
   ├─ 质量检测（Qwen3.5-VL）
   └─ 生成详情页模板

3. 文案生成阶段
   ├─ 多语言标题优化（SEO）
   ├─ 卖点提炼（5点描述）
   └─ 详情页文案

4. 上架阶段
   ├─ 多平台同步上架
   ├─ 价格策略设置
   └─ 广告投放设置

5. A/B测试阶段（7天）
   ├─ 监控CTR/CVR数据
   ├─ 自动识别爆款
   └─ 胜出变体保留

6. 爆款优化阶段
   ├─ 生成完整详情页（8-10张）
   ├─ 深度文案优化
   └─ 多平台放大
```

### 关键业务指标

| 阶段 | 指标 | 目标值 |
|------|------|--------|
| **测款** | 日测款数量 | 1,200个SKU |
| **主图** | CTR（点击率） | >5% 优秀 |
| **详情页** | CVR（转化率） | >3% 合格 |
| **爆款** | 爆款率 | 15% |
| **成本** | 单SKU测试成本 | 0.042元 |

## 🔧 技术亮点

### 1. 为什么选 FLUX.1 而不是 FLUX.2?

```
FLUX.2-dev:
├─ 显存需求: 54GB+ (单卡装不下)
├─ 质量: 更好
└─ 速度: 更慢

FLUX.1-dev + IPAdapter + ControlNet:
├─ 显存需求: 24GB (刚好)
├─ 质量: 92分（已足够）
└─ 速度: 更快

结论: FLUX.1 + 增强组件 = 最佳性价比
```

### 2. 为什么选 Qwen3.5 而不是 DeepSeek-R1?

```
Qwen3.5-35B-A3B:
├─ 原生多模态（图文视频统一）
├─ 201语言支持（覆盖所有市场）
├─ 推理速度快（无需思考链）
└─ 阿里电商基因（理解SKU、MOQ）

DeepSeek-R1:
├─ 纯文本（需要额外视觉模型）
├─ 主要中英文
├─ 需要思考链（慢）
└─ 通用模型（无电商优化）

结论: Qwen3.5 更适合电商场景
```

### 3. 为什么选 SGLang 而不是 vLLM?

```
SGLang:
├─ 吞吐量: 200-250 tokens/s
├─ RadixAttention: 自动KV缓存
└─ 原生JSON Schema支持

vLLM:
├─ 吞吐量: 150 tokens/s
├─ 无自动缓存
└─ 需要额外配置

提升: 29%性能提升
```
